# Agent Guide for hextant Development

## Package Overview

**hextant** is an R package for SCUBA-based hexbin plotting of single-cell sequencing data, reimplementing functionality from the schex package. It creates hexagonal binning visualizations of dimension reduction results (e.g., UMAP, PCA) to efficiently display density and other features of single-cell datasets.

**Key Dependencies:**
- `SCUBA` - Single-cell data handling (GitHub: amc-heme/SCUBA)
- `ggplot2` - Plotting framework
- `hexbin` - Hexagonal binning algorithms
- `dplyr` - Data manipulation
- `rlang` - Non-standard evaluation

**Domain Context:** Single-cell sequencing generates high-dimensional data. Dimension reduction techniques (UMAP, t-SNE, PCA) project this into 2D/3D space. This package bins cells into hexagons for efficient visualization when datasets contain millions of cells.

## Code Style & Conventions

### Indentation & Spacing
- Use **4 spaces** for indentation (no tabs)
- Space after commas: `c(1, 2)` not `c(1,2)`
- Space around operators: `x <- 5` not `x<-5`
- No space before `(` in function calls: `plot()` not `plot ()`
- Line length: Keep under 80-100 characters when reasonable

### Naming Conventions
- **Functions**: `snake_case` - `plot_hextant_density()`
- **Variables**: `snake_case` - `dimension_reduction`, `use_dims`
- **Internal functions**: Prefix with `.` - `.hextant_bin()`
- **Constants**: `UPPER_SNAKE_CASE` if needed
- **Arguments**: `snake_case` matching tidyverse style

### Function Structure

function_name <- function(
    required_param,
    optional_param = default_value,
    another_param = NULL) {

## Package-Specific Patterns

### SCUBA Integration
Use `SCUBA::fetch_reduction()` to extract dimension reduction coordinates:
```r
dr <- fetch_reduction(obj, reduction = dr, dims = use_dims)
# Returns matrix with only the requested dimensions (simplified vs base R object extraction)
```

#### SCUBA Data Retrieval Functions Reference

All SCUBA fetch functions return data with **cells as the primary dimension**, ensuring consistency across different data types. This is critical for proper data alignment and aggregation.

**fetch_reduction():**
- **Returns:** Matrix with cells as rows, dimensions as columns
- **Usage:** `dr <- fetch_reduction(obj, reduction = "UMAP", dims = c(1, 2))`
- **Extract:** `dr[, 1]` gets first dimension for all cells
- **Example dimensions:** 250 cells × 2 dimensions

**fetch_metadata():**
- **Returns:** Data.frame with cells as rows, metadata variables as columns
- **Usage:** `meta <- fetch_metadata(obj, vars = "cell_type")`
- **Extract:** `meta$cell_type` or `meta[["cell_type"]]`
- **Cell IDs:** Available as row names

**fetch_feature():**
- **Returns:** **Data.frame** with **cells as ROWS**, **features as COLUMNS**
- **Usage:** `expr <- fetch_feature(obj, features = "CD8A", assay = "RNA", layer = "data")`
- **Extract:** `expr[["CD8A"]]` or `expr[, 1]` (for single feature)
- **IMPORTANT:** Features are COLUMNS, not rows!
- **Common mistake:** Using `expr[1, ]` extracts one cell, not one feature

**Key Pattern:**
All three functions align by cell ordering, making it safe to combine data:
```r
# These all have the same cell ordering and length
dr <- fetch_reduction(obj, reduction = "UMAP", dims = c(1, 2))      # Matrix: n_cells × 2
meta <- fetch_metadata(obj, vars = "cell_type")                      # DF: n_cells × 1
expr <- fetch_feature(obj, features = "CD8A")                        # DF: n_cells × 1

# Can safely combine without cell ID matching (but validate lengths!)
length(dr[, 1]) == nrow(meta) == nrow(expr)  # Should be TRUE
```

### Hexbin Workflow Pattern
Standard pattern for creating hexagonal bins:
```r
# 1. Get dimension reduction coordinates
dr <- fetch_reduction(obj, reduction = "UMAP", dims = c(1, 2))

# 2. Calculate bounds
xbnds <- range(dr[, 1])
ybnds <- range(dr[, 2])

# 3. Create hexbin object
drhex <- hexbin(dr[, 1], dr[, 2], nbins, xbnds = xbnds, ybnds = ybnds, IDs = TRUE)

# 4. Extract cell assignments and coordinates
cID <- drhex@cID
coords <- hcell2xy(drhex)
```

### Metadata-Based Splitting with `split_by`
Use `SCUBA::fetch_metadata()` to access metadata and split plots by categorical variables:
```r
# Basic split by cell type
plot_hextant_density(obj, split_by = "cell_type")

# Control layout with ncol parameter
plot_hextant_density(obj, split_by = "treatment", ncol = 2)

# Use free scales for better per-facet contrast
plot_hextant_density(obj, split_by = "sample", scales = "free_y")
```

**Implementation pattern for metadata integration:**
- Use global binning across all cells (consistent bin positions)
- Each facet shows the same bins with group-specific cell counts
- Bins with zero cells for a group are excluded from that facet
- Handle NA values by converting to "NA" string for faceting
- Validate metadata variable exists before plotting
- Warn if metadata has >10 unique values (plot readability)

**Key helper function:**
```r
# .add_metadata_to_bins() expands hexbins by metadata groups
expanded_df <- .add_metadata_to_bins(obj, cID, hexbin_matrix, metadata_var, dr_name, use_dims)
# Returns data frame with one row per (bin, group) combination where count > 0
```

### Independent Density Scales with `scale_density`

When using `split_by`, you can choose between shared or independent color scales for density visualization:

**Shared scale (default):**
```r
# All facets use the same color scale (absolute cell counts)
plot_hextant_density(obj, split_by = "cell_type")
```
In this mode, colors represent absolute cell counts. If one group has 1000 cells and another has 100, the smaller group will appear mostly dark/low on the scale.

**Independent scales:**
```r
# Each facet has its own 0-1 rescaled color scale (relative density)
plot_hextant_density(obj, split_by = "cell_type", scale_density = TRUE)
```
In this mode, each facet's density values are rescaled to 0-1 independently. Both groups use the full color range, making within-group patterns easier to see.

**When to use independent scales (`scale_density = TRUE`):**
- Groups have very different cell counts (e.g., one group has 10x more cells)
- You want to see relative density patterns within each group
- You want better visual contrast in low-density groups
- Comparing spatial distribution patterns rather than absolute numbers

**When to use shared scales (default, `scale_density = FALSE`):**
- You want to compare absolute cell densities across groups
- All groups have similar cell counts
- You want to see which group has the most cells in each region
- Absolute cell numbers are important for interpretation

**Combining with position scales:**
```r
# Both independent density (color) and free y-axis (position) scales
plot_hextant_density(obj, split_by = "treatment", 
                      scale_density = TRUE, scales = "free_y", ncol = 2)
```

**Implementation details:**
- Rescales `number_of_cells` to 0-1 within each group
- Creates a `scaled_density` column for plotting
- Legend shows "Scaled Density" when `scale_density = TRUE`
- Only applies when `split_by` is used (warning issued otherwise)
- Handles edge cases: if all bins in a group have the same count, scaled to 0.5

### Feature Expression Plotting

Use `SCUBA::fetch_feature()` to access gene expression data for feature visualization:
```r
# Get expression for specific features
expr_data <- fetch_feature(obj, features = "CD8A", assay = "RNA", layer = "data")
# Returns data.frame: cells as rows (e.g., 250 cells), features as columns (1 feature)
# Feature name is the column name, cell IDs are row names
feature_vector <- as.numeric(expr_data[["CD8A"]])  # Extract feature column as numeric vector
```

**CRITICAL:** `fetch_feature()` returns a **data.frame** with:
- **Rows = cells** (one row per cell)
- **Columns = features** (one column per requested feature)
- Feature names as column names
- Cell IDs as row names

**Common mistake to avoid:**
```r
# WRONG: This extracts the FIRST CELL (one value only!)
feature_vector <- as.numeric(expr_data[1, ])

# CORRECT: This extracts the FEATURE COLUMN (all cells)
feature_vector <- as.numeric(expr_data[[feature_name]])
# or
feature_vector <- as.numeric(expr_data[, 1])  # If only one feature requested
```

**Basic feature plotting:**
```r
# Plot mean expression in hexbins
plot_hextant_feature(obj, feature = "CD8A")

# Use different aggregation methods
plot_hextant_feature(obj, feature = "CD8A", action = "median")
plot_hextant_feature(obj, feature = "CD8A", action = "sum")

# Split by metadata
plot_hextant_feature(obj, feature = "CD8A", split_by = "cell_type")
```

**Implementation pattern for feature aggregation:**
- Extract expression values as numeric vector (one value per cell)
- Use `tapply()` to aggregate by bin ID (from cID)
- Supported aggregation methods:
  - `"mean"`: Average expression per bin (default)
  - `"median"`: Median expression per bin (robust to outliers)
  - `"sum"`: Total expression per bin (useful for absolute levels)
- Match aggregated values to bin coordinates using bin ID mapping
- Handle NA values with `na.rm = TRUE`

**split_by with features:**
- Aggregate expression separately within each (bin × group) combination
- Only show bins where the group has cells with expression data
- Use `.add_metadata_to_bins_with_feature()` helper for expansion
- Returns data frame with columns: x, y, feature_value, metadata_var
- Example:
  ```r
  # Each facet shows expression for that group only
  plot_hextant_feature(obj, feature = "CD8A", split_by = "cell_type", 
                        ncol = 2, scales = "free")
  ```

**Key helper functions:**
```r
# .calculate_feature_per_bin() aggregates expression by bin
aggregated <- .calculate_feature_per_bin(feature_values, cID, action = "mean")
# Returns numeric vector of aggregated values (one per bin)

# .add_metadata_to_bins_with_feature() expands bins with per-group expression
expanded_df <- .add_metadata_to_bins_with_feature(
    obj, cID, hexbin_matrix, bin_ids, feature_values, 
    action, metadata_var, dr_name, use_dims
)
# Returns data frame with one row per (bin × group) with aggregated expression
```

## Error Handling & Validation

### NULL Handling for Optional Parameters
Provide defaults when parameters are NULL:
```r
if (is.null(title)) {
    title <- "Density"
}

if (is.null(xlab)) {
    xlab <- "x"
}
```

### Parameter Validation
Add checks for critical parameters:
```r
# Validate numeric ranges
stopifnot(nbins > 0)
stopifnot(length(use_dims) == 2)

# Validate object types
if (!inherits(obj, "Seurat") && !inherits(obj, "SingleCellExperiment")) {
    stop("obj must be a SCUBA-supported single-cell object")
}
```

### Informative Error Messages
```r
if (missing(dimension_reduction)) {
    stop("dimension_reduction must be specified (e.g., 'UMAP', 'PCA')")
}
```
